Privacy-Safe Security (k-Anonymity)

How it works: When you run a query, the application transforms the entry into a cryptographically secure SHA-1 hash. Only the first 5 characters of this hash are sent to the HaveIBeenPwned service. The API server returns a list of matching hashes sharing that prefix. The application then performs the comparison locally on your machine.

Result: Neither the third-party API nor our server ever sees the plain password, and it is impossible to reconstruct it from the 5-character prefix. Your credentials remain private.
